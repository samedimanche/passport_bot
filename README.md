# 📄 Passport Scanner Bot

Асинхронный Telegram-бот для распознавания данных из фотографии паспорта (внутреннего или заграничного, любой страны). Бот приватный — доступ только у ограниченного списка пользователей.

## Возможности

- Приём фото документа в Telegram и автоматическое распознавание:
  - Фамилия, Имя
  - Отчество (если удалось распознать)
  - Дата рождения
  - Дата начала и дата окончания действия документа
  - Номер паспорта / документа
  - Страна выпуска паспорта
  - Гражданство
  - Место рождения
- Разбор **машиночитаемой зоны (MRZ)** по стандарту ICAO 9303 с проверкой контрольных цифр — это делает основные поля (ФИО, даты, номер документа) надёжными почти в 100% случаев на чётком фото.
- Дополнительные поля (место рождения, дата выдачи) распознаются обычным OCR по тексту страницы — точность ниже, рекомендуется проверять вручную.
- Whitelist доступа по Telegram `user_id` — пользоваться могут только заранее указанные аккаунты.
- Полностью асинхронная архитектура (aiogram 3 + aiohttp), работа через webhook — минимальная нагрузка на бесплатный хостинг.
- Тяжёлая операция (OCR) выполняется в отдельном потоке, не блокируя обработку остальных сообщений.

## Стек

- Python 3.11
- [aiogram 3](https://docs.aiogram.dev/) — асинхронный Telegram Bot Framework
- [Tesseract OCR](https://github.com/tesseract-ocr/tesseract) — бесплатное офлайн-распознавание текста
- OpenCV — предобработка изображения для повышения качества OCR
- Docker — для установки Tesseract с языковыми пакетами на хостинге

## Структура проекта

```
passport_bot/
├── main.py           # точка входа: aiohttp-сервер + webhook aiogram
├── config.py          # переменные окружения (токен, доступ, языки OCR)
├── handlers.py         # обработчики сообщений бота
├── ocr_utils.py        # предобработка фото, распознавание MRZ и текста
├── requirements.txt      # зависимости Python
├── Dockerfile          # образ с установленным Tesseract
├── .env.example         # пример переменных окружения
└── .gitignore
```

## Переменные окружения

| Переменная      | Описание                                                              | Пример                                |
|-----------------|------------------------------------------------------------------------|----------------------------------------|
| `BOT_TOKEN`     | Токен бота от [@BotFather](https://t.me/BotFather)                     | `123456:ABC-DEF...`                    |
| `ALLOWED_IDS`   | Telegram `user_id` пользователей с доступом, через запятую             | `111111111,222222222`                  |
| `WEBHOOK_HOST`  | Публичный URL сервиса на хостинге                                      | `https://passport-bot.onrender.com`    |
| `OCR_LANGS`     | Языки Tesseract через `+` (опционально)                                | `rus+eng+deu+fra+spa+ita+ukr`          |
| `PORT`          | Порт сервера (подставляется хостингом автоматически)                   | `10000`                                |

Свой `user_id` можно узнать у бота [@userinfobot](https://t.me/userinfobot).

## Локальный запуск

```bash
git clone <ссылка на репозиторий>
cd passport_bot
cp .env.example .env   # заполните значения
pip install -r requirements.txt
python main.py
```

> Для локального запуска потребуется установленный Tesseract OCR с нужными языковыми пакетами (в проде это делает `Dockerfile`).

## Деплой (Render.com, бесплатно, без консоли)

1. Загрузить проект в GitHub-репозиторий.
2. На [render.com](https://render.com) → **New → Web Service** → подключить репозиторий.
3. Render автоматически определит `Dockerfile` (Environment: Docker).
4. Instance Type: **Free**.
5. Задать переменные окружения (`BOT_TOKEN`, `ALLOWED_IDS`, `WEBHOOK_HOST`).
6. После первого деплоя скопировать выданный Render URL в `WEBHOOK_HOST` и передеплоить — бот сам зарегистрирует webhook при старте.
7. (Рекомендуется) Настроить бесплатный пинг через [UptimeRobot](https://uptimerobot.com) на `/`, чтобы сервис не «засыпал».

## Ограничения точности

MRZ (машиночитаемая зона) не содержит места рождения и даты выдачи документа — эти поля стандартом не предусмотрены ни в одном паспорте мира. Они извлекаются обычным распознаванием текста страницы и требуют визуальной проверки, особенно если язык документа не входит в список `OCR_LANGS`.

## Безопасность

- Доступ к боту ограничен списком `ALLOWED_IDS` — все остальные пользователи получают отказ.
- Путь webhook включает токен бота, что исключает обращение к нему посторонних сервисов.
- Фотографии не сохраняются на диск — обрабатываются только в памяти процесса.
